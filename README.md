# Универсальный скрипт расширения диска (LVM и обычные разделы)

Данный скрипт автоматически расширяет раздел диска на всё доступное свободное место (или на указанный размер) с поддержкой LVM и обычных файловых систем (ext2/3/4, XFS). Работает с MBR и GPT, умеет перемещать swap-раздел, если целевой раздел не последний.

## Возможности

- Автоматическое определение типа разметки (MBR / GPT)
- Проверка наличия свободного неразмеченного места после последнего раздела
- Расширение последнего раздела с помощью growpart
- Если целевой раздел не последний, а следующий за ним — swap:
  - Предлагает расширить swap (без перемещения)
  - Или автоматически удалить swap, расширить целевой раздел, затем создать swap в конце диска (с обновлением /etc/fstab)
- Поддержка LVM: после расширения раздела выполняет pvresize и lvextend -r для всех LV (кроме swap)
- Поддержка обычных разделов: автоматическое расширение файловой системы (resize2fs / xfs_growfs)
- Интерактивный выбор диска и раздела, цветовой вывод подсказок
- Проверка установки необходимых утилит (parted, gdisk, growpart) и автоматическая установка через apt/yum/dnf при отсутствии

## Требования

- Linux с bash
- Права root (запуск через sudo)
- Виртуальный диск должен быть предварительно увеличен в гипервизоре (VMware, KVM, VirtualBox и т.п.)
- На диске должен быть хотя бы один раздел, который можно расширить

## Установка

1. Скачайте скрипт:
   ```bash
   wget https://raw.githubusercontent.com/ваш-репозиторий/expand_disk.sh
   ```
   или создайте файл вручную.

2. Дайте права на выполнение:
   ```bash
   chmod +x expand_disk.sh
   ```

## Использование

Запустите от root:

```bash
sudo ./expand_disk.sh
```

Скрипт последовательно запросит:
- Имя диска (например, sda, sdb, nvme0n1)
- Номер раздела для расширения (по умолчанию последний, но можно указать любой)
- Если раздел не последний и за ним находится swap, предложит два варианта действий

Все действия логируются в stderr, интерактивный ввод осуществляется в stdin.

## Алгоритм работы

1. Проверка прав root
2. Проверка/установка пакетов parted, gdisk, growpart
3. Выбор диска и проверка наличия свободного места после последнего раздела
4. Выбор раздела для расширения
5. Если раздел не последний и следующий — swap, предложить действия
6. Расширение раздела через growpart
7. Обновление таблицы разделов (partprobe)
8. Если LVM: pvresize и lvextend -r для всех LV (кроме swap)
9. Если обычный раздел: расширение ФС (resize2fs / xfs_growfs)
10. Вывод итогового состояния диска

## Обработка ошибок

- Если диск не увеличен и нет свободного места, скрипт сообщит об этом и завершится
- Если growpart возвращает код 2 (NOCHANGE), скрипт выводит причину и выходит
- При отсутствии необходимых утилит скрипт попытается установить их автоматически
- Перед перемещением swap создаётся резервная копия /etc/fstab
